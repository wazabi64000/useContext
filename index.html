Guide React useContext - Gestion Utilisateur

Objectif : comprendre et implémenter useContext dans un projet React (Vite) pour gérer un utilisateur global.

1. Pourquoi useContext ?

useContext permet de partager des données globales (comme un utilisateur connecté) sans passer par les props.

  • Évite le "prop drilling"
  • Centralise l’état global
  • Facilite la maintenance

2. Initialisation projet Vite

npm create vite@latest cd projet npm install npm run dev
---

3. Exemple 1 — Création du Context

// src/context/UserContext.jsx import { createContext } from "react"; /* createContext permet de créer un contexte global ici on initialise avec null (aucun utilisateur) */ export const UserContext = createContext(null);

Explication :

  • createContext crée un espace global
  • null = pas encore connecté
---

4. Exemple 2 — Provider global

// src/context/UserProvider.jsx import { useState } from "react"; import { UserContext } from "./UserContext"; /* Provider = composant qui va encapsuler l'application */ export const UserProvider = ({ children }) => { // état utilisateur global const [user, setUser] = useState(null); return ( /* value = données accessibles partout */ <UserContext.Provider value={{ user, setUser }}> {children} </UserContext.Provider> ); };

Points clés :

  • useState stocke l’utilisateur
  • Provider rend accessible user et setUser
---

5. Exemple 3 — Injection dans App

// main.jsx import React from "react"; import ReactDOM from "react-dom/client"; import App from "./App"; import { UserProvider } from "./context/UserProvider"; /* On enveloppe toute l'application avec le Provider */ ReactDOM.createRoot(document.getElementById("root")).render( <UserProvider> <App /> </UserProvider> );

Tous les composants enfants peuvent maintenant accéder au contexte.

---

6. Exemple 4 — Utilisation du contexte

// src/components/Profile.jsx import { useContext } from "react"; import { UserContext } from "../context/UserContext"; const Profile = () => { /* useContext permet de récupérer les données globales */ const { user } = useContext(UserContext); return ( <div> {user ? ( <p>Bienvenue {user.name}</p> ) : ( <p>Non connecté</p> )} </div> ); }; export default Profile;
---

7. Exemple 5 — Connexion utilisateur (interaction)

// src/components/Login.jsx import { useContext } from "react"; import { UserContext } from "../context/UserContext"; const Login = () => { const { setUser } = useContext(UserContext); const handleLogin = () => { /* Simulation connexion utilisateur */ setUser({ name: "John Doe", email: "john@mail.com" }); }; return ( <button onClick={handleLogin}> Se connecter </button> ); }; export default Login;

Résultat :

  • clic → utilisateur défini globalement
  • tous les composants reçoivent la mise à jour
---

8. Bonnes pratiques

  • Séparer Context et Provider
  • Limiter le nombre de contexts
  • Utiliser des hooks personnalisés
  • Ne pas surcharger le context
---

9. Architecture recommandée

src/ ├── context/ │ ├── UserContext.jsx │ ├── UserProvider.jsx │ ├── components/ │ ├── Login.jsx │ ├── Profile.jsx
---

10. Conclusion

useContext est une solution simple et efficace pour gérer l’état global dans des applications React de taille moyenne.
