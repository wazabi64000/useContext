Guide React useContext - Gestion Utilisateur

Objectif : apprendre à gérer un utilisateur global dans React avec useContext, étape par étape, pour débutants complets.

1. Théorie : pourquoi useContext ?

Dans une application React, il arrive souvent que plusieurs composants aient besoin de connaître l’état d’un utilisateur (par exemple : connecté ou non, nom, email). Sans useContext, il faudrait passer ces informations depuis un composant parent à chaque enfant (prop drilling), ce qui devient compliqué dans des applications larges.

useContext permet :

  • de partager des données globales facilement
  • de simplifier le code et éviter de passer les props partout
  • de centraliser l’état (user) et les fonctions pour le modifier

2. Initialisation du projet Vite

Pour commencer un projet React moderne, on utilise Vite :

// 1. Crée un projet vite npm create vite@latest mon-projet-react // 2. Se déplacer dans le dossier du projet cd mon-projet-react // 3. Installer les dépendances npm install // 4. Lancer le serveur de développement npm run dev

Explication :

  • npm install : installe les packages nécessaires
  • npm run dev : lance le projet dans le navigateur

3. Exemple 1 : créer un Context

On commence par créer un contexte global pour l’utilisateur.

// src/context/UserContext.jsx import { createContext } from "react"; // createContext crée un "espace global" // Ici, on initialise le contexte avec null (aucun utilisateur connecté) export const UserContext = createContext(null);

Explication :

  • On importe createContext depuis React
  • On crée un contexte appelé UserContext
  • La valeur initiale est null car l’utilisateur n’est pas encore connecté

4. Exemple 2 : créer le Provider

Le Provider enveloppe notre application et permet de partager l’état utilisateur.

// src/context/UserProvider.jsx import { useState } from "react"; import { UserContext } from "./UserContext"; // Composant Provider export const UserProvider = ({ children }) => { // useState pour stocker les informations de l'utilisateur const [user, setUser] = useState(null); return ( // rend user et setUser accessibles dans toute l'application <UserContext.Provider value={{ user, setUser }}> {children} // Tous les composants enfants </UserContext.Provider> ); };

Explication détaillée :

  • useState : permet de créer une variable d’état reactive
  • user : l’état actuel (informations utilisateur)
  • setUser : fonction pour mettre à jour l’utilisateur
  • value={{ user, setUser }} : données accessibles par tous les composants qui consomment ce contexte
  • {'{children}'} : représente tous les composants React enveloppés par le Provider

5. Exemple 3 : envelopper l'application

On enveloppe notre application avec le Provider pour que tous les composants puissent accéder au contexte.

// src/main.jsx import React from "react"; import ReactDOM from "react-dom/client"; import App from "./App"; import { UserProvider } from "./context/UserProvider"; // On enveloppe App avec UserProvider ReactDOM.createRoot(document.getElementById("root")).render( <UserProvider> <App /> </UserProvider> );

Explication :

  • Tout ce qui est dans <App /> pourra accéder au contexte utilisateur
  • Pas besoin de passer user ou setUser via props

6. Exemple 4 : utiliser le contexte dans un composant

Maintenant, un composant peut consommer le contexte pour afficher le nom de l’utilisateur.

// src/components/Profile.jsx import { useContext } from "react"; import { UserContext } from "../context/UserContext"; const Profile = () => { // On récupère les données du contexte const { user } = useContext(UserContext); return ( <div> {user ? ( <p>Bienvenue {user.name}</p> ) : ( <p>Non connecté</p> )} </div> ); }; export default Profile;

Explication :

  • useContext(UserContext) : récupère l’état global
  • Si user existe → affiche le nom
  • Sinon → affiche "Non connecté"

7. Exemple 5 : mettre à jour l'utilisateur (connexion)

On crée un bouton pour simuler la connexion d’un utilisateur.

// src/components/Login.jsx import { useContext } from "react"; import { UserContext } from "../context/UserContext"; const Login = () => { const { setUser } = useContext(UserContext); // on récupère la fonction pour mettre à jour user const handleLogin = () => { // Simulation de connexion : on définit un utilisateur setUser({ name: "John Doe", email: "john@mail.com" }); }; return ( <button onClick={handleLogin}> Se connecter </button> ); }; export default Login;

Explications :

  • setUser est appelée pour définir les informations utilisateur
  • Tous les composants consommant le contexte se mettent à jour automatiquement
  • Exemple simple mais montre le flux global de données

8. Bonnes pratiques pour débutants

  • Séparer Context et Provider pour plus de clarté
  • Ne pas surcharger un contexte avec trop de données
  • Créer des hooks personnalisés si nécessaire pour simplifier l'accès au contexte
  • Utiliser useContext pour des états globaux simples (user, thème, paramètres)
  • Pour des états complexes, envisager Redux ou Zustand
